## Preflight: TARGET_PROJECT 解析与校验

在开始 TDD 前，先执行：

1. 读取控制面板根目录 `AGENTS.md` 中的 `TARGET_PROJECT`
2. 将路径解析为实际代码仓库根目录
3. 校验目录存在，且看起来像可执行项目根目录
4. 向用户回显最终解析路径
5. 后续所有接口定义、测试编写、测试运行、实现与重构都在 `TARGET_PROJECT` 中进行

若 `TARGET_PROJECT` 缺失或无效，立即停止并要求先修正配置。

对指定的功能模块执行 TDD 开发：

1. **定义接口 (SCAFFOLD)**
   - 先定义数据结构和函数签名
   - 实现体留空或抛出"未实现"异常

2. **写失败的测试 (RED)**
   - 在实现之前编写测试用例
   - 覆盖：正常路径、边界情况、错误情况
   - 使用项目技术栈推荐的测试模式（参考 `AGENTS.md` Coding Style 和 `.antigravity/examples/`）

3. **运行测试，确认失败**
   - 运行项目的测试命令（由 `/genesis` 确定）
   - 确认测试因"未实现"而失败（不是因为编译错误）

4. **写最小实现 (GREEN)**
   - 只写让测试通过的最少代码
   - 不做优化、不做多余功能

5. **运行测试，确认通过**
   - 运行项目的测试命令
   - 全部绿色

6. **重构 (REFACTOR)**
   - 提取常量、改善命名、消除重复
   - 保持测试绿色

7. **验证覆盖率**
   - 运行项目的覆盖率检测命令（由 `/genesis` 确定）
   - 目标：80%+，核心逻辑 100%

重复步骤 2-7 直到模块完成。
